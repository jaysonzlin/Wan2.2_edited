## Post-static research backlog

The following work is intentionally deferred. Each item is tied to a static claim above; do not start GPU work until the prerequisite evidence is available.

1. **Checkpoint metadata audit — prerequisite for architecture claims.** Read the checkpoint’s `config.json` and weight index without loading weights. Confirm `model_type`, DiT `in_dim`, DiT `out_dim`, text dimensions, and the tensor names/shapes expected by `WanModel.from_pretrained(checkpoint_dir)` (`wan/textimage2video.py:102-109`). Update the tensor ledger only from this metadata.
2. **Primary-source reconciliation — prerequisite for training claims.** Read the Wan2.2 technical material and record, separately, the training target, noise/data interpolation, loss weighting, data composition, and reported evaluations. Link each claim to a primary source and compare it to the scheduler interface at `wan/utils/fm_solvers_unipc.py:281-350`; discrepancies become ledger entries rather than assumptions.
3. **VAE round-trip experiment — tests the representation boundary.** Hypothesis: deterministic use of `mu` provides stable reconstruction, while the `4×16×16` bottleneck creates measurable reconstruction loss. Encode/decode representative clips, measure per-frame reconstruction metrics, and inspect whether errors concentrate in motion or texture. The code basis is deterministic `mu` return at `wan/modules/vae2_2.py:783-810` and decode at `wan/modules/vae2_2.py:812-839`.
4. **I2V protected-slice ablation — tests unification.** Hypothesis: first-frame identity relies on both the re-imposition `latent = (1-M)z_img + Mlatent` and the zero-timestep patch map; removing either should weaken initial-frame preservation differently. Compare full I2V with (a) no re-imposition and (b) no zero-timestep map, holding prompt, image, seed, solver, and steps fixed. The intervention sites are `wan/textimage2video.py:573-578` and `wan/textimage2video.py:591-598`.
5. **CFG and shift factorial sweep — tests sampling sensitivity.** Hypothesis: CFG modifies the vector field by the exact affine rule at `wan/textimage2video.py:380-386`, while shift changes the distribution of integration points at `wan/utils/fm_solvers_unipc.py:184-213`; these parameters may interact. Sweep guide scales and shifts over a fixed prompt/image set, using identical seeds and a predeclared scorecard for prompt alignment, temporal coherence, motion, and artifacts.
6. **Solver comparison — tests numerical rather than model differences.** Hypothesis: with identical initial noise, prompt contexts, CFG, and shifted schedule, observed differences between UniPC and DPM++ arise from the solver update rules rather than different neural weights. Compare the `unipc` and `dpm++` branches selected at `wan/textimage2video.py:335-354`.
7. **Activation and attention instrumentation — tests architectural roles.** Hypothesis: time modulation and global 3D self-attention distribute temporal/coarse layout information differently from text cross-attention. Instrument block inputs/outputs, modulation norms, and attention summaries across layers; map results back to the block equation at `wan/modules/model.py:237-258`. Do not interpret attention scores as causal explanations without intervention tests.
